\section{KVP10k Dataset}
\label{sec:kvp10k}

The KVP10k dataset \cite{kvp10k} is a large-scale benchmark for \kvp{} extraction from business documents. It consists of 48,280 training samples and 5,255 test samples, spanning diverse document types including invoices, forms, and receipts.

\subsection{Data Collection}
The dataset was collected by IBM Research from real-world business documents, with manual annotation of bounding boxes and key-value relationships.

\subsection{Annotation Format}
Each sample contains:
\begin{itemize}
    \item \textbf{Image}: Document scan/photo
    \item \textbf{Annotations}: List of bounding boxes with:
    \begin{itemize}
        \item \texttt{coordinates}: List of $\{x, y\}$ points defining polygon
        \item \texttt{label}: Annotation type (Text, identifier, etc.)
        \item \texttt{attributes.Linking}: Reference to linked annotation ID
    \end{itemize}
\end{itemize}

\section{Layout Clustering Analysis}
\label{sec:layout_clustering}

We perform unsupervised clustering to identify recurring document layout types in the dataset. This provides a compact way to stratify evaluation in later stages and to check whether model performance varies across layout regimes.

\subsection{K-means Clustering}
\label{subsec:kmeans}

We apply K-means clustering \cite{kmeans} on standardized layout features. To select the number of clusters, we sweep $k \in \{2,\dots,10\}$ and evaluate multiple complementary criteria: (i) Silhouette score (primary), (ii) Davies--Bouldin index, (iii) Calinski--Harabasz score, and (iv) the inertia curve (elbow). We use a fixed random seed and standardization to make the selection reproducible.

Figure~\ref{fig:stage2_k_selection_and_distribution} summarizes the selection outcome. Across the considered range, the criteria consistently favor a small number of well-separated clusters. In particular, the silhouette score peaks at \textbf{0.763} for $k=2$ (0.7627), and we therefore choose \textbf{$k=2$}.

\textbf{Results:} Two distinct layout types were identified on the annotated KVP10k training pages ($n=25{,}692$):

\section{Key-Value Link Analysis}
\label{sec:kv_analysis}

We analyze spatial relationships between linked \kvps{} to inform model design.

\subsection{Spatial Proximity}

For each key-value pair with linking annotation, we compute the Euclidean distance between bounding box centers:

\begin{equation}
d_{kv} = \sqrt{(x_v - x_k)^2 + (y_v - y_k)^2}
\end{equation}

where $(x_k, y_k)$ and $(x_v, y_v)$ are the normalized center coordinates of the key and value boxes.

\textbf{Results} (200-sample subset, 755 links):
\begin{itemize}
    \item Mean distance: $\mu_d = 0.10$
    \item Median distance: $\tilde{d} = 0.08$
    \item 90th percentile: $d_{90} = 0.15$
\end{itemize}

\textbf{Interpretation:} Most \kvps{} are spatially proximate (within 0.15 normalized units), suggesting that spatial features are crucial for accurate linking.

\section{Summary}
\label{sec:dataset_summary}

Key findings from dataset analysis:
\begin{enumerate}
    \item Two distinct layout types with varying density
    \item Strong spatial proximity between linked \kvps{}
    \item High annotation quality with consistent linking information
    \item Layout features capture 82.7\% variance in 2 dimensions
\end{enumerate}

These insights inform our model design, particularly the importance of spatial encoding in the linker module.
