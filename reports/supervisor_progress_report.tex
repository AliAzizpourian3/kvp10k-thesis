\section{Thesis Recap}
% ============================================================

This thesis investigates \textbf{key-value pair (KVP) extraction} from semi-structured documents (invoices, receipts, forms) using IBM's \textbf{KVP10k} benchmark dataset (ICDAR 2024, Naparstek et al.).
The KVP10k paper argued that encoder-based models such as LayoutLMv3 have limitations for direct KVP linking, and therefore evaluated only a generative LMDX-style baseline rather than empirically benchmarking LayoutLMv3 on KVP10k.

This thesis directly tests that claim by combining \textbf{layout-aware modelling} with \textbf{data-centric analysis}.
The research goals from the proposal are:

\begin{enumerate}[nosep]
  \item \textbf{Evaluate Layout-Aware Modelling:} The KVP10k paper (Naparstek et al., 2024) argued, without direct empirical evidence, that encoder-based models such as LayoutLMv3 are insufficient for KVP linking, and consequently did not benchmark them on the task. This thesis empirically tests that claim by extending LayoutLMv3 with a lightweight Biaffine Linker module for explicit key--value relationship extraction, and evaluating it directly on the KVP10k benchmark.
  \item \textbf{Analyze and Categorize Document Layouts:} Apply unsupervised clustering to group documents by structural similarity and explore model performance across layout types.
  \item \textbf{Assess Robustness and Generalization:} Investigate model behaviour under layout distortions and compare OCR-based with OCR-free approaches in zero-shot and few-shot settings.
  \item \textbf{Derive Analytical Insights:} Examine learning trends, ablation effects, and error patterns.
\end{enumerate}

The project is organized into five stages (Figure~\ref{fig:pipeline}).
\textbf{All stages are now complete.}

\section{Completed Work (Stages 0--4)}
% ============================================================

\subsection{Stage 0: Evaluation Protocol}
The evaluation protocol is adopted directly from the KVP10k paper (Naparstek et al., ICDAR 2024):
\begin{itemize}[nosep]
  \item \textbf{Text-only:} Normalized Edit Distance (NED) $< 0.2$ between predicted and ground-truth text.
  \item \textbf{Text + Location:} NED $< 0.2$ \emph{and} bounding-box IoU $> 0.3$.
\end{itemize}

\noindent
The IoU threshold of $0.3$ is adopted directly from Naparstek et al.\ (2024), who chose this value to permit minor spatial imprecision in predicted bounding boxes while still ensuring that the prediction is located in the correct spatial region of the page.
This is the same threshold used in their published evaluation, so all results in this thesis are directly comparable to the KVP10k paper baseline.

\subsection{Stage 1: Data Ingestion \& Preparation}

The raw KVP10k dataset required substantial preparation before model training:

\begin{enumerate}[nosep, leftmargin=1.5em]
  \item \textbf{Deduplication:} Pages grouped by \texttt{hash\_name}; richest annotation selected. Raw 48{,}280 train rows $\to$ 9{,}656 unique pages; 5{,}255 test rows $\to$ 1{,}051 unique pages.
  \item \textbf{PDF download \& validation:} Source PDFs downloaded and verified per unique page.
  \item \textbf{Text extraction:} PyMuPDF used to extract words and bounding boxes. Pages without text layers discarded.
  \item \textbf{Annotation fusion:} Words matched against annotation bounding boxes to assign Key/Value/Other labels and reconstruct KVP link supervision.
  \item \textbf{LMDX formatting:} Each word encoded as \texttt{word x1|y1|x2|y2} in normalised coordinates.
\end{enumerate}

\subsection{Stage 4: Discriminative Model (LayoutLMv3 + Biaffine Linker) --- Complete}

\subsubsection*{Architecture}

LayoutLMv3-base (125M parameters) with two task heads:
\begin{enumerate}[nosep]
  \item \textbf{Entity Classifier:} Linear layer classifying each token as Key, Value, or Other.
  \item \textbf{Biaffine Linker:} Pairs key spans with value spans using semantic (scaled dot-product) and spatial (8-feature MLP) scores. Trained with binary cross-entropy loss on a key\,$\times$\,value adjacency matrix.
\end{enumerate}

\subsubsection*{Results}

The full story of Stage~4 experiments is as follows:

\paragraph{Stage 4a --- Entity Pre-training.}
LayoutLMv3 trained with entity classifier only (linker disabled) achieves \textbf{entity F1 = 0.847}, confirming strong token-level key/value classification.

\paragraph{Experiment 1 --- Token-Level Linking (V1).}
Activating the biaffine linker at the token level: each key token scored against each value token, producing a dense $\sim$10{,}000-candidate matrix with $\sim$450:1 neg:pos ratio. Result: \textbf{link F1 = 0.008} despite entity F1 = 0.841. The $\lambda$ sweep over \{0.5, 1.0, 2.0\} confirms link F1 stays near zero regardless of hyperparameter --- the failure is structural, not a tuning issue.

\paragraph{Experiment 2 --- Span-Level Linking (V2).}
Grouping contiguous same-label tokens into spans before linking reduces candidates to $\sim$225 per page with $\sim$5:1 neg:pos ratio. Yet initial V2 result: \textbf{link F1 = 0.007} --- still near zero. This prompted a systematic diagnostic investigation.

\paragraph{Experiments D1--D3 --- Diagnostic Analysis.}
Three root causes were identified and corrected:
\begin{itemize}[nosep]
  \item \textbf{Bug 1 (Missing Teacher Forcing):} Linker received predicted entity labels (noisy) instead of ground-truth labels during training, misaligning span boundaries with link labels.
  \item \textbf{Bug 2 (Bounding Box Filter):} A strict bbox validity filter $(x_1 < x_2) \wedge (y_1 < y_2)$ rejected 49\% of valid key tokens and 46\% of valid value tokens (LayoutLMv3 normalises to a $[0,1000]$ grid, producing many zero-width tokens).
  \item \textbf{Bug 3 (Link Label Cross-Product Explosion):} Label generation created a positive entry for every key-word $\times$ value-word combination within a KVP (instead of one per KVP pair), inflating the link label matrix to a near-uniform positive signal --- effectively providing no supervision.
\end{itemize}

\paragraph{Experiment 3 --- Corrected Model (V4).}
Applying all three fixes and retraining (30 epochs, $\lambda = 5.0$, early stop at epoch 13):

\begin{center}
\begin{tabular}{lcccc}
\toprule
\textbf{Model} & \textbf{Entity F1} & \textbf{Link F1} & \textbf{Neg:Pos} & \textbf{Note} \\
\midrule
V1 (token-level) & 0.841 & 0.008 & $\sim$450:1 & Token imbalance \\
V2 (span-level)  & 0.847 & 0.007 & $\sim$5:1   & Pipeline bugs \\
\textbf{V4 (corrected)} & \textbf{0.827} & \textbf{0.673} & $\sim$5:1 & Bugs fixed \\
KVP10k paper baseline & ---   & 0.659 & ---         & Published result \\
\bottomrule
\end{tabular}
\end{center}

The corrected V4 model achieves \textbf{link F1 = 0.673}, matching the KVP10k published baseline (0.659 text-only) with a 125M-parameter discriminative model.
Crucially, entity F1 = 0.827 reflects the discriminative model's ability to classify only tokens that actually exist in the document, \textbf{eliminating the hallucination problem} that plagued the generative Mistral baseline on sparse layouts.

\section{Technical Challenges Encountered}
% ============================================================

\begin{enumerate}[leftmargin=1.5em]

\item \textbf{HPC Offline Compute Nodes.}
  FAU HPC compute nodes have no internet access. All models had to be pre-downloaded and cached; all \texttt{from\_pretrained()} calls required \texttt{local\_files\_only=True}.

\item \textbf{CUDA / PyTorch / Quantization Compatibility.}
  Resolving version conflicts between PyTorch, Transformers, and bitsandbytes for 4-bit QLoRA on CUDA~13.0.

\item \textbf{Data Preparation Pipeline.}
  Broken/inaccessible source PDFs ($\sim$45\% of pages), conflicting duplicate annotations, and reproducing IBM's workflow with PyMuPDF rather than their internal OCR tooling.

\item \textbf{Stage 4 Training Stability.}
  Early runs suffered NaN losses from unclamped biaffine logits. Fixed with logit clamping to $[-20, 20]$ and teacher forcing.

\item \textbf{Three Compounding Data Pipeline Bugs.}
  The diagnostic work (Experiments D1--D3) required significant engineering to isolate: the bugs interacted in a way that made each individually hard to attribute. Fixing all three at once in V4 was the turning point.

\end{enumerate}

\section{Literature Review Findings}
% ============================================================

\begin{itemize}[leftmargin=1.5em]
  \item \textbf{SPADE} (Hwang et al., ACL Findings 2021): spatial dependency parsing, directly inspired the spatial MLP in the Stage~4 linker.
  \item \textbf{BROS} (Hong et al., AAAI 2022): BERT-based with 2D relative position encoding and a token prediction linking head.
  \item \textbf{LMDX} (Perot et al., 2023): generative LLM approach using LMDX-formatted prompts. The approach evaluated in the KVP10k paper; published F1 = 0.643 is the primary comparison point.
  \item \textbf{LayoutLMv3} (Huang et al., ACM MM 2022): joint text + layout + image pre-training; entity F1 = 0.91 on FUNSD. Adding a biaffine linker for KVP extraction is the novel contribution of this thesis.
\end{itemize}
